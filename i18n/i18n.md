# How to Add a Language

Claude Hangar supports internationalized documentation. This guide explains how to add a new language translation.

## Directory Structure

```
i18n/
├── i18n.md             # This guide
├── de/                 # German
│   ├── README.md       # Translated README
│   └── docs/           # Translated documentation
│       ├── getting-started.md
│       └── ...
├── fr/                 # French (example)
│   ├── README.md
│   └── docs/
└── ja/                 # Japanese (example)
    ├── README.md
    └── docs/
```

Each language gets its own directory using the [ISO 639-1](https://en.wikipedia.org/wiki/List_of_ISO_639-1_codes) two-letter language code.

## What Needs Translating

### Required

1. **README.md** — The main project README. This is the entry point for users in that language.

### Optional (but recommended)

2. **docs/getting-started.md** — Quick start guide
3. **docs/configuration.md** — Configuration reference
4. **docs/hooks.md** — Hook documentation
5. **docs/skills.md** — Skill documentation

### Do NOT Translate

- Code comments (keep in English)
- Variable names and identifiers
- CLAUDE.md files (these are Claude Code instructions, always English)
- CI workflow files
- JSON schema files
- Test scripts

## Adding a New Language

### 1. Create the Directory

```bash
mkdir -p i18n/{lang}
mkdir -p i18n/{lang}/docs
```

Replace `{lang}` with the ISO 639-1 code (e.g., `fr`, `ja`, `es`, `pt`).

### 2. Translate the README

Start with `README.md` as the most important file:

```bash
cp README.md i18n/{lang}/README.md
# Then translate the content
```

Add a link back to the English README at the top:

```markdown
> [English version](../../README.md)
```

### 3. Translate Documentation

Copy and translate the docs you want to cover:

```bash
cp docs/getting-started.md i18n/{lang}/docs/getting-started.md
# Then translate
```

### 4. Link from the Main README

Add your language to the language selector in the root `README.md`:

```markdown
**Languages:** [English](README.md) | [Deutsch](i18n/de/README.md) | [Your Language](i18n/{lang}/README.md)
```

## Keeping Translations in Sync

Translations can drift out of date as the English source evolves. To manage this:

1. **Version tag** — Add a comment at the top of each translated file noting which English version it was translated from:
   ```markdown
   <!-- Translated from: English README @ commit abc1234 (2026-03-15) -->
   ```

2. **Review on release** — When a new version is released, check if translated files need updating by comparing the English source against the tagged commit.

3. **Partial translations are OK** — It is better to have an accurate partial translation than a complete but outdated one. Mark untranslated sections with:
   ```markdown
   > **Note:** This section has not been translated yet. See the [English version](../../docs/section.md).
   ```

## Translation Quality

- Use natural, idiomatic language (not word-for-word translation)
- Keep technical terms in English where the translated term is uncommon (e.g., "Hook", "Skill", "Stack")
- Preserve all links and code examples
- Test that all relative links work from the `i18n/{lang}/` directory
